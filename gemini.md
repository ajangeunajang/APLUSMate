# A+Mate 프로젝트 지침

이 문서는 A+Mate 프로젝트의 기술 스택, 코딩 스타일 및 아키텍처에 대한 지침을 제공합니다.

## 1. 프로젝트 개요

A+Mate는 사용자가 PDF 파일을 업로드하고 관리하며 학습에 도움을 주는 웹 애플리케이션입니다.

- **프레임워크**: [Next.js](https://nextjs.org/) (React)
- **언어**: [TypeScript](https://www.typescriptlang.org/)
- **스타일링**: [Tailwind CSS](https://tailwindcss.com/)
- **인증**: [NextAuth.js](https://next-auth.js.org/)
- **패키지 매니저**: `yarn`

## 2. 시작하기

프로젝트를 로컬 환경에서 설정하고 실행하는 방법입니다.

1.  **의존성 설치**:

    ```bash
    yarn install
    ```

2.  **개발 서버 실행**:

    ```bash
    yarn dev
    ```

    애플리케이션은 `http://localhost:3000`에서 실행됩니다.

## 3. 코딩 스타일

일관성 있고 가독성 높은 코드를 유지하기 위해 다음 지침을 따릅니다.

- **언어**: 모든 새 코드는 TypeScript로 작성합니다.
- **포맷팅**:
  - 들여쓰기는 공백 2칸을 사용합니다.
  - `eslint`와 `prettier` (권장)를 사용하여 코드 스타일을 일관되게 유지합니다.
  - 실행 전 `yarn lint`를 통해 린트 오류를 확인합니다.
- **컴포넌트**:
  - 함수형 컴포넌트와 Hooks (`useState`, `useEffect` 등)를 사용합니다.
  - 컴포넌트 파일명은 파스칼 케이스 (`PascalCase.tsx`)를 사용합니다.
- **타이핑**:
  - `interface` 또는 `type`을 사용하여 props와 상태의 타입을 명확하게 정의합니다.
  - `any` 타입 사용을 최소화합니다.
- **API 통신**:
  - 클라이언트 측에서는 `fetch` API를 사용하여 백엔드와 통신합니다.
  - API 요청 및 응답 데이터에 대한 타입을 정의합니다.
- **주석**:
  - 복잡한 로직이나 다른 개발자가 이해하기 어려운 부분에 대해 간결한 주석을 추가합니다.
  - 코드 자체로 설명이 가능한 부분에는 주석을 자제합니다.

## 4. 폴더 구조

프로젝트의 주요 폴더 구조는 다음과 같습니다.

- **`/app`**: Next.js 13+의 앱 라우터 관련 파일이 위치합니다.
  - **`/app/api`**: 백엔드 API 라우트 핸들러가 위치합니다.
  - **`/app/components`**: 재사용 가능한 리액트 컴포넌트가 위치합니다.
  - **`/app/pdf`**: PDF 뷰어 페이지가 위치합니다.
- **`/public`**: 이미지, 폰트 등 정적 에셋 파일이 위치합니다.
- **`/types`**: 전역적으로 사용되는 TypeScript 타입 정의가 위치합니다.

## 5. API 라우트

- API 라우트는 `/app/api` 디렉토리 내에 파일 기반 라우팅으로 생성됩니다.
- 각 API는 특정 기능을 담당하며, 주로 외부 백엔드 서버와 통신하는 역할을 합니다.
- API 핸들러 함수는 `NextRequest`와 `NextResponse` 객체를 사용하여 요청과 응답을 처리합니다.

## 6. 인증

- `NextAuth.js`를 사용하여 사용자 인증을 처리합니다.
- 현재 지원하는 인증 방식은 **자격증명(credentials)** 및 **Google OAuth**입니다.
- 인증 관련 로직은 `/app/api/auth/[...nextauth]/route.ts` 파일에 정의되어 있습니다.
- 세션 정보는 `useSession` 훅을 통해 클라이언트 컴포넌트에서 접근할 수 있습니다.
